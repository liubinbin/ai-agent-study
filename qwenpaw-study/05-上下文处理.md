# 上下文处理

由于模型的上下文大小有限制，所以我们在单次请求的上下文需要控制在一定的大小。本章节主要关注请求上下文的问题。目前 QwenPaw 上下文已经抽象到 ContextManager。

## 初始化
1. 在 src/qwenpaw/config/config.py AgentsRunningConfig 中 L857 context_manager_backend:str=Field(default="light") 默认 light 类型
2. 在 src/qwenpaw/agents/context/light_context_manager.py 中注册 LightContextManager 为 light。
3. 在 src/qwenpaw/agents/react_agent.py QwenPawAgent 初始时，从 context_manager 通过  get_agent_context 方法获取的 context 写入到 memory。get_agent_context 方法的功能是获取对话保存路径。
4. 在 src/qwenpaw/agents/react_agent.py QwenPawAgent 初始时，通过 self._register_hooks() 方法若有配置 context_manager，则注册四个 hook ，包括 pre_reply、pre_reasoning、post_acting、post_reply 四个。

hook 执行顺序详见 [1]

## pre_reply
在执行 reply 之前，若 auto_memory_search_config.enabled 开启，调用 memory_manager.retrieve 从 memory 获取。

## pre_reasoning
在 reasoning 之前，需要控制往大模型发送的内容，所以此处需要做压缩。
此方法内，执行如下几步
1. 先判断目前是否需要做压缩，根据 message 大小。
2. 对消息做压缩。
3. 做完压缩后对压缩的消息做标记。

## post_acting
在执行完 tool 后，处理返回结果过大的情况

## post_reply
在处理完请求后，取出最新的几个用户输入，把这个输入的总结任务加入到队列中。

## 总结
上下文管理就是在恰当的时候对 memory 的内容做总结压缩，在 QwenPaw 中，使用了 hook 能力，通过在几个阶段加入插入代码，实现这个管理能力。
