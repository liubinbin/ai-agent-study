# 上下文处理

由于模型的上下文大小有限制，因此需要将单次请求的上下文控制在一定范围内。本章节主要关注请求上下文的问题。目前 QwenPaw 的上下文管理已抽象到 ContextManager。

## 初始化

1. 在 src/qwenpaw/config/config.py 的 AgentsRunningConfig 中，L857 定义了 `context_manager_backend: str = Field(default="light")`，默认为 light 类型。
2. 在 src/qwenpaw/agents/context/light_context_manager.py 中注册 LightContextManager 为 light。
3. 在 src/qwenpaw/agents/react_agent.py 的 QwenPawAgent 初始化时，通过 context_manager 的 `get_agent_context` 方法获取 context，并写入 memory。`get_agent_context` 方法的功能是获取对话保存路径。
4. 在 QwenPawAgent 初始化时，通过 `self._register_hooks()` 方法，若配置了 context_manager，则注册四个 hook：pre_reply、pre_reasoning、post_acting、post_reply。

hook 执行顺序详见 [1]。

## pre_reply

在执行 reply 之前，若 `auto_memory_search_config.enabled` 为 true，则调用 `memory_manager.retrieve` 从 memory 中获取信息。

## pre_reasoning

在 reasoning 之前，需要控制发送给大模型的内容大小，因此此处需要做压缩。

此方法内执行如下几步：

1. 根据 message 大小判断目前是否需要压缩。
2. 对消息做压缩。
3. 压缩完成后对压缩后的消息做标记。

## post_acting

在执行完 tool 后，处理返回结果过大的情况。

## post_reply

在处理完请求后，取出最新的几个用户输入，将这些输入的总结任务加入队列。

## 总结

上下文管理就是在恰当的时机对 memory 的内容进行总结和压缩。在 QwenPaw 中，利用 hook 机制，在几个关键阶段插入代码，从而实现这一管理能力。
