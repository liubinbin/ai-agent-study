# 多智能体隔离

QwenPaw 支持多个智能体，可以为每个智能体配置的自己个性化 md 文件（AGENTS.md、SOUL.md、PROFILE.md 及 Mermoy 文件），详见资料[1]。

## AgentRunner 
之前所有分析是从 AgentRunner 的 query_handler 开始。
在这个 AgentRunner 里有 agent_id，有 workspace_dir，有个自己的智能体信息文件。

## 启动初始化
QwenPaw 使用 FastAPI 对外提供 http 服务，所以从 FastApi 开始。
1. 创建 FastAPI 对赋值给 app。
2. 初始化一个 DynamicMultiAgentRunner 赋值给 runner，然后使用 runner 初始化一个 AgentApp 对象 agent_app。
3. 在的 AgentApp 的 _add_endpoint_router 方法中，把 stream_query 的包装器加入路由中。
4. 在 AgentApp 内，_internal_framework_lifespan 把根据 stream 参数把 stream_query 或 query 方法加入路由，此方法是为了其他协议兼容使用。
5. 然后 agent_app 的路由加入到 app 中。这样 fastapi 就可以对外提供 agent 的方法了。

## DynamicMultiAgentRunner 分发请求
请求进入 DynamicMultiAgentRunner 的 stream_query 方法，在 /src/qwenpaw/app/_app.py 中的 def stream_query 方法。
1. 通过 _get_workspace 方法根据请求获得 workspace。
2. 从 workspace 获取 runner，调用 runner 的 stream_query 方法。

## _get_workspace 方法
初始化方法：在 fastapi 初始化时，会初始化一个 MultiAgentManager，然后传递给 DynamicMultiAgentRunner。
1. 先通过 get_current_agent_id 获取 agent_id。
2. 然后通过 _multi_agent_manager 的 get_agent 获取 agent。

## get_agent 方法
1. 若在缓存中有，则获取后返回。
2. 或新建会使用一个 event 放入 map，防止多个请求进来初始化。
3. 在初始化时，通过 workspace 的 start 方法完成所有组件的初始化，workspace 的初始化参数来自配置文件，看起来这个方法耗时很长，
