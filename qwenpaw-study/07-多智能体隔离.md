# 多智能体隔离

QwenPaw 支持多个智能体，可以为每个智能体配置个性化的 md 文件（AGENTS.md、SOUL.md、PROFILE.md 及 Memory 文件），详见资料 [1]。

## AgentRunner

之前的所有分析都是从 AgentRunner 的 query_handler 开始的。

在这个 AgentRunner 里有 agent_id、workspace_dir 以及智能体自身的信息文件。

## 启动初始化

QwenPaw 使用 FastAPI 对外提供 HTTP 服务，因此从 FastAPI 开始分析。

1. 创建 FastAPI 实例并赋值给 app。
2. 初始化一个 DynamicMultiAgentRunner 并赋值给 runner，然后使用 runner 初始化一个 AgentApp 对象 agent_app。
3. 在 AgentApp 的 `_add_endpoint_router` 方法中，将 stream_query 的包装器加入路由。
4. 在 AgentApp 内，`_internal_framework_lifespan` 根据 stream 参数将 stream_query 或 query 方法加入路由，此方法用于兼容其他协议。
5. 然后将 agent_app 的路由加入到 app 中。这样 FastAPI 就可以对外提供 agent 的方法了。

## DynamicMultiAgentRunner 分发请求

请求进入 DynamicMultiAgentRunner 的 stream_query 方法，即 /src/qwenpaw/app/_app.py 中的 stream_query 方法。

1. 通过 `_get_workspace` 方法根据请求获取 workspace。
2. 从 workspace 获取 runner，并调用其 stream_query 方法。

## _get_workspace 方法

初始化方法：在 FastAPI 初始化时，会初始化一个 MultiAgentManager，并将其传递给 DynamicMultiAgentRunner。

1. 先通过 `get_current_agent_id` 获取 agent_id。
2. 然后通过 `_multi_agent_manager` 的 `get_agent` 方法获取 agent。

## get_agent 方法

1. 若缓存中已有，则直接获取并返回。
2. 若需新建，则使用一个 event 放入 map 中，防止多个请求同时进入初始化流程。
3. 在初始化时，通过 workspace 的 start 方法完成所有组件的初始化。workspace 的初始化参数来自配置文件，看起来这个方法耗时很长。
