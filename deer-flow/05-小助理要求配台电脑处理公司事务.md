## 小助理要求配台电脑处理公司事务

小助理说自己的电脑是要打游戏的，不能拿来处理公司的事情，上级说：好的，给你一个 sandbox 用于处理公司事务。

## sandbox 流程
   1. 注册 middleware：在 make_lead_agent 方法中 create_agent 会传入 middleware，其中会加入 SandboxMiddleware，具体见 backend/packages/harness/deerflow/agents/middlewares/tool_error_middleware.py L78。
   2. 初始化 sandbox 并加入 state：在 SandboxMiddlebox 的 before_agent 方法，会去获取一个 sandbox，在完成获取后，将 sandbox_id 加入到 state 中，详见 backend/packages/harness/deerflow/sandbox/middleware.py L52。
   3. 获取 sandbox 并执行命令：以 bash 为例，先通过 ensure_sandbox_initialized 方法获取 sandbox，获取方法就是从 state 中获取 sandbox_id，然后去 get_sandbox_provider 获取 sandbox，获取到 sandbox 后，直接调用 sandbox 的 execute_command 执行命令。详见 backend/packages/harness/deerflow/sandbox/tools.py L761。 
   4. 释放 sandbox：在 SandboxMiddleware 的 after_agent 方法，会去获取 sandbox，然后调用 release 方法释放它，详见 backend/packages/harness/deerflow/sandbox/middleware.py L68。

## sandbox 框架设计
 provider 基类提供 acquire、get、release 方法，处理申请、获取和释放，详见 backend/packages/harness/deerflow/sandbox/sandbox_provider.py L8。
 sandbox 基类提供 execute_commond、read_file 等方法，详见 backend/packages/harness/deerflow/sandbox/sandbox.py L4。
 
 在开发中，主要用到的是 local 和 aio 两种类型的 sandbox，local 是直接在宿主机上运行 sandbox 代码，aio 是在隔离的 docker 容器里运行 sandbox 代码。
