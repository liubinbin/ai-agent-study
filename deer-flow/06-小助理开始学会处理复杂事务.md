# 小助理开始学会处理复杂事务

当我们在遇到复杂问题时，都是需要在顶层把任务分解为子任务，然后在每个子任务中去列出TODO，依次或并发处理完所有子任务后，最后在顶层把所有的内容都整合在一起，完成这个复杂的任务。
从中我们会发现，要完成复杂问题，有几样是需要注意的，1、问题不能过于复杂，子任务不能太多，否则最后的整合做不了。2、子任务的 TODO 需要记住执行完。3、在子任务中需要保持能处理完，子任务不能做成一个复杂任务。

## subagent 模式
在 make_lead_agent 时，若客户端设置 subagent_enabled 为 true，会开启 subagent 模式。在创建 lead agent 时，会做几件事，一、会在 tools 中加入 SUBAGENT_TOOLS 的 task_tool，这个 tool 能处理里复杂问题，二、在 prompt 提示不能分解太多的任务，三、添加 SubagentLimitMiddleware 这个中间件，做并发控制。

## SubagentLimitMiddleware
在 SubagentLimitMiddleware 的 after_model 方法中，若发现 tool 调用过多，会把部分 tool 给丢弃掉，目前 subagent 的限制 MAX_CONCURRENT_SUBAGENTS 默认为 3。

## SummarizationMiddleware
SummarizationMiddleware 的实现，deerflow 并没有自己定制一个，而使用了 langchain 的，但主动设置了几个参数，详见 backend/packages/harness/deerflow/agents/lead_agent/agent.py L41 的 _create_summarization_middleware 方法。配置内容见 config.yaml 里的 summarization 章节。

## TodoMiddleware
TodoMiddleware 这个中间件是扩展了 langchain 的 TodoListMiddleware。
TodoListMiddleware 这个中间件原为在开启 plan mode 时，使用 write_todos 字段来更新各个 todo 的状态，另外在 state 中加了 todo 字段。
TodoMiddleware 这个中间件会在 write_todos 没有时，todo 字段还有时，加入 prompt 提示还有 todo 要做，这种情况会发生在 SummarizationMiddleware 意外把一些 todo 去除时发生，所以这个 中间件需要排在 SummarizationMiddleware 后面。

## DanglingToolCallMiddleware
针对 tool_calls 没有对应 ToolMessage 的情况，在对应位置添加一条 tool 被打断的 ToolMessage，可以让发送给的模型的消息是完整的。
其中 tool_calls 是大模型生成的，要调用某个工具的记录； ToolMessage 是工具执行结果。两者通过 id 关联。
