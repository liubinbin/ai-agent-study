# 小助理学会了更多技能

外部技能主要是通过三种 tools 和 skill 扩展实现的。

## tools
在前面的章节中，create_agent 方法会传入 tools 字段，tools 列表包括两个部分，一部分是通过 backend/packages/harness/deerflow/tools/tools.py L35 的 get_available_tools 方法获取，另一部分是 setup_agent，读取了 SOUL.md 用于获取个性化信息。
从代码可以看到 tools 主要由四个部分组成，分别是loaded_tools、builtin_tools、mcp_tools、acp_tools。
  - loaded_tools：读取 config.yaml 的 tools 下的工具，从配置文件获取包和类名后，通过 resolve_variable 反射获取 LangChain 的 tool 实例，工具列表主要是 web 工具，文件操作工具等。
  - builtin_tools：两个必备工具，包括呈现产物给用户、询问用户更多信息，若启用了子agent，还会添加 general-purpose、bash 两个子 agent 库。若启用了 mcp search，则还会添加 tool_search。
  - mcp_tools：从 mcp server 获取的 tools 列表。
  - acp_tools：读取配置文件中的 acp_agents，获取到的通过 acp 扩展的工具，acp详见[1]。

## skill
与上面的 tools 不同， skill 是通过 prompt 加载进去的。
在 backend/packages/harness/deerflow/agents/lead_agent/prompt.py L480 apply_prompt_template 方法中，通过 get_skills_prompt_section 获取 skill 目录下所有 skill 的 name 和 description 及 skill 目录。（从中，我们可以看到 name 和 description 需要更加简练）

## skill 目录处理
在配置文件中，skills.container_path 目录默认是 /mnt/skills。
若在 aio sandbox 时，创建容器时会做特殊的 mount，包括 skill 的挂载，详见 backend/packages/harness/deerflow/community/aio_sandbox/aio_sandbox_provider.py L191 _get_extra_mounts 方法。

## 扩展 skill 
只需将符合语法的 skill 文件放入到对应的 skill 目录，系统会自动读取目录，并在请求中使用上这些 skill。
