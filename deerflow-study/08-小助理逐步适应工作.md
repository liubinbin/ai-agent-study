# 小助理逐步适应工作

小助理不停处理任务过程中，学习到了一些上级在工作上特殊的要求，例如数据要准确，每到季度最后一个月要做报表等，这些要求都要记下来，以便未来能更好地满足上级的工作要求。在 deerflow 中，是通过 memory 体系完成这个记忆工作。

## 配置文件
有一个 memory.json，专门记录这些需要记录的信息。
包括 user 记录用户信息，history 记录一些历史上的任务倾向，facts 记录一些提到的事实。

## MemoryMiddleware
从 state 获取所有 message，拿出用户和 AI 返回的消息，把消息扔入队列中

## system_prompt
在 backend/packages/harness/deerflow/agents/lead_agent/prompt.py L480 apply_prompt_template 方法获取 system_prompt 时，也包含了 memory 中获取到的信息。

## 并发控制
memory 更新的并发控制时通过队列解耦。
入队列：MemoryMiddleware 把单次请求过滤出的消息包装成一个记录写入到队列中。
出队列：在 /backend/packages/harness/deerflow/agents/memory/queue.py L87 的 _process_queue 处理队列的记录，调用 updater.update_memory 方法做实际的更新。

## update_memory 流程
1. 读取memory文件，获取 memory 数据
2. 通过系统提示词 MEMORY_UPDATE_PROMPT 让 LLM 生成新的 memory
3. 将 LLM 生成 memory 信息与原有的合并，对于 facts 内容，多加了 confidence 过低时过滤掉，个数过多时排序取前几个
4. 将内容写入写回文件中
