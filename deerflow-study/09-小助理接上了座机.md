# 小助理接上了座机

小助理为了更方便的接收上级的任务，接了一个座机，这样他就不需要每次上级过来找他再分配任务了，只需要打个电话就行，方便多了。

## 配置
关于 channels 的配置在 config.yaml 的 channels，包含 langgraph_url、gateway_url、session，以及配置的第三方，如feishu、slack、telegram。

## app 启动
1.在 /backend/app/gateway/app.py 的 create_app 方法中，整个 app 启动通过 lifespan 传给 fastapi。
2.在 /backend/app/gateway/app.py 的 lifespan 方法中，调用 start_channel_service 方法启动 channel 服务。
3.在 /backend/app/channels/service.py 的 ChannelService 启动时，会初始化 一个 消息总线 MessageBus 和一个channel 管理服务 ChannelManager，MessageBus 服务消息的交互，ChannelManager 负责管理 agent 与总线上消息的交互。
4.对于每个 channel，例如 feishu，在 /backend/app/channels/feishu.py L106 会把自己处理消息的方法在数据总线加一个回调的监听。

## 消息总线
分为几个部分，处理入数据和出数据。
1.有一个队列的保存入数据，数据来自各个 channel 对第三方的监听，内容都是客户端在第三方输入的问题。
2.有一个数组保存外部消息的监听列表，在收到消息时，会回调每个监听方法。

## ChannelManager
1.监听总线数据，这次数据来自客户从各个。
2.获取 langgraph_url 获取 agent client。
3.将消息传递给 agent。
4.不停接受来自 agent 的消息，拼装成 OutboundMessage。
5.publish 这条消息，实际操作是通过回调列表不停回调注册的监听者。

##  消息总流程
以feishu为例，有两条主链路。
agent接收消息：飞书 -> 总线 -> channnelManager -> agent
agent回复消息：agent -> channnelManager -> 回调列表 -> 飞书
